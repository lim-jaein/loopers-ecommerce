## US-01 상품 목록 조회
[유저 스토리]
- 사용자는 상품 목록을 조회할 수 있다.

[기능 흐름]
1. 로그인 사용자는 상품 목록을 조회할 수 있다.
2. 사용자는 특정 브랜드의 상품을 필터링하여 조회할 수 있다. 
3. 사용자는 정렬 조건(신상품순, 낮은 가격순, 높은 좋아요순)으로 조회할 수 있다.
4. 상품 목록 조회 시 각 상품의 총 좋아요 수가 포함된다.
5. 상품 목록은 페이징 처리되어 반환된다. 

## US-02 상품 상세 조회
[유저 스토리]
- 사용자는 상품 상세 정보를 조회할 수 있다.

[기능 흐름]
1. 로그인 사용자는 상품 ID로 특정 상품의 상세 정보를 조회할 수 있다. 
2. 상품이 존재하면 상품의 총 좋아요 수가 포함된 상품 상세 정보가 반환된다. 
   - 상품이 존재하지 않으면 404 Not Found 응답을 반환한다.

## US-03 브랜드 정보 조회
[유저 스토리]
- 사용자는 브랜드 정보를 조회할 수 있다.

[기능 흐름]
1. 로그인 사용자는 브랜드 ID로 특정 브랜드 정보를 조회할 수 있다. 
2. 브랜드가 존재하면 브랜드 정보가 반환된다. 
   - 브랜드가 존재하지 않으면 404 Not Found 응답을 반환한다.

## US-04 좋아요 등록/취소
[유저 스토리]
- 사용자는 각 상품에 한 번만 좋아요를 누를 수 있다.
- 상품에 대한 좋아요 등록/취소 기능은 멱등하게 동작한다.
- 사용자는 자신이 좋아요 한 상품 목록을 조회할 수 있다.

[기능 흐름]
1. 로그인 사용자는 특정 상품에 좋아요 등록 또는 취소를 누를 수 있다. 
   - 해당 상품이 존재하지 않으면 404 Not Found 응답을 반환한다. 
2. 상품이 존재하면, 요청에 따라 다음을 수행한다.
   - 좋아요 등록 요청 시, 좋아요 상태가 아니라면 등록되고 상품의 좋아요 수가 증가한다.
   - 취소 요청 시, 이미 좋아요한 상태일 때만 취소되고 상품의 좋아요 수가 감소한다.
3. 사용자는 자신이 좋아요한 상품 목록을 조회할 수 있다.

## US-05 주문 요청
[유저 스토리]
- 사용자는 주문을 요청할 수 있다.

[기능 흐름]
1. 로그인 사용자는 주문할 상품과 각 상품의 수량을 선택한다.
2. 사용자가 주문을 클릭하면 주문 요청이 전송된다.
3. 시스템은 아래 케이스를 모두 만족할 경우 상품 재고와 사용자의 포인트를 차감하고 주문을 생성한다.
   - 주문 상품이 모두 존재하고, 각 상품의 재고가 주문 수량 이상이고, 사용자의 포인트가 총 주문 금액 이상이어야 한다.
     - 주문상태가 CREATED -> PAID로 변경된다.
   - 각 조건을 만족하지 못할 경우 각각 404 Not Found, 400 Bad Request, 400 Bad Request 응답을 반환한다.
     - 주문상태가 CREATED -> CANCELED로 변경된다.
4. 생성된 주문 정보를 외부 시스템으로 전송한다.
   - 전송이 실패하면 502 Bad Gateway 응답을 반환한다.

## US-06 주문 목록/상세 조회
[유저 스토리]
- 사용자는 자신의 주문 목록을 조회할 수 있다.
- 사용자는 자신의 주문 상세 정보를 조회할 수 있다.

[기능 흐름]
1. 로그인 사용자는 자신의 주문 목록을 조회할 수 있다. 
2. 사용자는 특정 주문 ID로 자신의 주문 상세 정보를 조회할 수 있다. 
   - 주문이 존재하면 주문 상세 정보가 반환된다. 
   - 주문이 존재하지 않으면 404 Not Found 응답을 반환한다.
